Scroll Fade

Cards fade and slide up as they enter the viewport. Powered by CSS animation-timeline: view() with an IntersectionObserver fallback.

What is scroll-driven animation?

CSS scroll-driven animations let you tie animation progress directly to a scroll container's scroll position. The view() timeline maps the element's progress through the viewport — 0% when it first appears, 100% when it's fully visible.

Zero JavaScript

On supported browsers (Chrome 115+, Edge 115+), the entire animation runs on the compositor thread with no JavaScript overhead. The @supports wrapper ensures the CSS only activates where supported.

Staggered Delays

Each card has a progressive delay via nth-child selectors, creating a cascading entrance effect. The delays are purely CSS — no inline styles or JS needed.

animation-range Control

The animation-range: entry 0% cover 30% setting means the animation starts when the element enters the viewport and completes by the time 30% of it is visible. This ensures cards finish fading before they reach center screen.

Cross-Browser Fallback

For Firefox, Safari, and other browsers that don't yet support animation-timeline: view(), an IntersectionObserver detects when cards enter the viewport and applies a .visible class to trigger CSS transitions.

Reduced Motion Support

When a user has prefers-reduced-motion: reduce enabled, all animations are disabled and cards are shown in their final state immediately. This respects accessibility preferences without breaking the layout.

Performance First

Scroll-driven animations run on the compositor thread, meaning they stay smooth even during heavy JavaScript execution or layout thrashing. The result is 60fps+ animations with zero effort.
